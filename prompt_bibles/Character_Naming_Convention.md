# CHARACTER NAMING CONVENTION — ADDENDUM TO CHARACTER BIBLE
### (Locks the naming pattern so any new fruit/vegetable character gets an on-brand name automatically, instead of being invented ad hoc per script.)

---

## THE PATTERN

Every character name is built the same way:

**[Recognizable syllable or sound from the fruit/vegetable name] + [affectionate diminutive suffix: -y / -o / -ie / -er]**

This does three jobs at once:
1. Keeps names easy for a TTS engine to pronounce cleanly in both English and [DIALOGUE_LANGUAGE].
2. Keeps names instantly "readable" as that fruit/vegetable, even to a 5-year-old viewer.
3. Keeps the tone consistently warm/cute — never a literal or clinical name.

## RULE FOR NAMING A BRAND-NEW CHARACTER

1. Take the most recognizable 1–2 syllable chunk of the fruit/vegetable's name.
2. Add a diminutive suffix (-y, -o, -ie, -er) unless the fruit's real name is already short and cute (Mango, Kiwi — use as-is).
3. Say it out loud — if it's awkward for a voice actor or TTS engine to hit cleanly, adjust the suffix.
4. Check it doesn't collide with an existing character name in the Bible.
5. Once decided, add the character to `Fruit_Character_Bible.md` (look), `Fruit_Character_Acting_Guide.md` (movement), and `Fruit_Character_Voice_Bible.md` (voice) in the same table format as existing entries — never leave a character in only one of the three files, or consistency breaks the moment they appear in a new script.

**Quick gender/role note:** the Bible currently skews male-coded for schemer/elder roles and female-coded for warmth/empathy roles. If you want to diversify future characters (e.g. a female schemer, a male gentle-heart type), that's a deliberate creative choice — just lock it the same way once decided, don't leave it ambiguous per-script.
