## AUTO-GENERATION RULES (for a brand-new location not yet in this library)

When a script needs a location that isn't listed above, build it using this checklist so it matches the established world style, then ADD it to this file:

1. **Anchor to the existing palette** — warm ochre/terracotta for village-life scenes, cool blue-grey for "status/wealth" scenes (Ozzy/Coco spaces), warm gold for emotional/festival scenes.
2. **Always specify**: shot type, 4–6 concrete props (not vague — "a table" is wrong, "a low wooden table with a single glowing oil lamp" is right), time of day, and the closing quality clause block below.
3. **Always end with the same closing block** so render style stays locked:
   `vibrant saturated colors, storybook atmosphere, no characters in frame, pure background plate, high detail render`
4. **Never put characters in the background plate** — characters are always composited in separately at Phase 5/6 so the same background file can be reused across multiple scenes and shots.
5. File name = location name, no spaces, no extension, PascalCase (matches the character file-naming convention).
