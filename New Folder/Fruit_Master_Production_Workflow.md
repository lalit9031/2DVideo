# FRUIT CARTOON CHANNEL — MASTER PRODUCTION WORKFLOW
### (The complete step-by-step guide combining the 7-Phase Pipeline + the 4 Consistency Bibles into one repeatable process for every future video)

---

## YOUR FULL ASSET LIBRARY (keep these 4 files open/handy for every video)

| File | Locks down |
|---|---|
| `Fruit_Character_Bible.md` | How each character LOOKS |
| `Fruit_Character_Acting_Guide.md` | How each character MOVES |
| `Fruit_Character_Voice_Bible.md` | How each character SOUNDS |
| `Fruit_Environment_Bible.md` | How the WORLD looks |

These never change between videos. Only the **story** changes each time.

---

## THE FULL WORKFLOW — STEP BY STEP

### STEP 1 — Confirm dialogue language
Decide the language for character dialogue (e.g. Hindi). Everything else (prompts, file names, tables) stays in English.
*Output: [DIALOGUE_LANGUAGE] locked for this video.*

---

### STEP 2 — Pick or generate a topic (Phase 1)
Generate/select 10 viral topic ideas using the standard formula:
`[Character A] Betrayed [Character B] | Fruits Cartoon Story`
Choose one topic to develop.
*Output: 1 locked topic with title, conflict, moral, characters, viral score.*

---

### STEP 3 — Write the dialogue-only script (Phase 2)
Write the full script using the 4-Act structure (Village life → Conflict → Consequence → Redemption → Closing), 18–30 scenes, max 6 sec/scene, dialogue in [DIALOGUE_LANGUAGE], action notes in English.
*Output: Locked script with total scenes + runtime.*

---

### STEP 4 — Pull character designs from the Character Bible (Phase 3, automated)
For every character in the script:
1. Open `Fruit_Character_Bible.md`
2. Find their entry (e.g. Appy, Ozzy, Berry)
3. Copy their **LOCKED IMAGE PROMPT** exactly
4. Only edit the outfit line if this specific story requires a costume change (e.g. wedding outfit, work uniform)

**Do NOT re-describe a character from scratch. Ever.** If a character is brand new (not yet in the Bible), design them once, then ADD them to the Bible so all future scripts can reuse them too.

*Output: One locked reference image prompt per character in this script.*

---

### STEP 5 — Pull environments from the Environment Bible (Phase 4, automated)
Scan the script for every location used.
1. Open `Fruit_Environment_Bible.md`
2. For each location, check if it's already in the library (Village Home, Farm, City Street, etc.)
3. If it exists → copy the **LOCKED PROMPT** directly
4. If it's new → use the **AUTO-GENERATION RULES** section to build it in the same style, then add it to the Bible for future reuse

*Output: One locked background prompt per unique environment in this script.*

---

### STEP 6 — Build scene-by-scene image prompts (Phase 5)
For each scene, combine:
- The **environment file name** (from Step 5)
- The **character file name(s)** present (from Step 4)
- Shot type (wide / medium / close-up, per the Shot Guide — e.g. betrayal = close-up with dark rim lighting)
- The specific pose/expression for that moment

*Output: A full storyboard — one image prompt per scene, referencing locked file names only.*

---

### STEP 7 — Build video prompts using all 4 bibles together (Phase 6)
This is where everything merges. For each scene's video prompt, layer in:

1. **Environment Bible** → background + ambient motion (leaves rustling, lanterns flickering)
2. **Character Bible** → visual appearance reference (via file name)
3. **Acting Guide** → movement style + signature gesture matched to the emotional beat
4. **Voice Bible** → locked voice profile (pitch, tone, pacing, accent) for the dialogue line
5. Apply the **ONE SPEAKER AT A TIME** rule — only one mouth moves at a time, other characters stay still and listening
6. Add BGM mood cue + ambient sound + transition tags

**Combined prompt formula:**
```
[Environment Bible background + ambient motion], SPEAKER: [FileName] ([Acting Guide movement style + signature gesture]) — mouth moving in lip-sync — [Voice Bible locked voice profile] saying "[line in DIALOGUE_LANGUAGE]" — other characters still and listening, BGM: [mood cue], AMBIENT: [sound], mood: [overall], high quality 3D animation render
```

*Output: One fully-merged video generation prompt per scene, using all 4 bibles at once.*

---

### STEP 8 — Generate viral metadata (Phase 7)
Titles (A/B/C), description with timestamps, tags, pinned comment, thumbnail brief — in English, with Hindi description/comment if dialogue language isn't English.
*Output: Complete upload-ready metadata package.*

---

### STEP 9 — Assemble and publish
Use the production checklist:
- [ ] Clips assembled in script order
- [ ] BGM mixed under dialogue (one speaker at a time confirmed)
- [ ] Outro card added
- [ ] Thumbnail created from the recommended emotional scene
- [ ] All metadata pasted into YouTube Studio
- [ ] Pinned comment posted within 10 minutes of upload

---

## THE AUTOMATION LOOP (for every video after this one)

```
NEW TOPIC IDEA
      ↓
Write script (Phase 2)
      ↓
Check Character Bible → reuse existing characters / add new ones once
      ↓
Check Environment Bible → reuse existing locations / add new ones once
      ↓
Build scene images (pull file names, no re-describing)
      ↓
Build video prompts (merge Character + Acting + Voice + Environment Bibles)
      ↓
Generate metadata
      ↓
Publish
      ↓
Bibles grow slightly (only if new characters/locations were added) → next video repeats loop
```

Over time, your 4 Bible files become a complete, ever-growing "show universe" — and every new video gets faster to produce because fewer and fewer NEW elements need to be created from scratch.

---

## GOLDEN RULE

**Story changes every video. Look, movement, voice, and world do not — unless the Bibles are deliberately updated.**
This single rule is what makes a channel feel professionally consistent instead of AI-generated and random.
